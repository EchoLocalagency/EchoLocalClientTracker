  <action>
Create `scripts/seo_engine/discovery.py` extending the directory_audit.py pattern with budget-gated Brave Search.

Script structure:
1. CLI interface: `python3 -m scripts.seo_engine.discovery --client-slug mr-green-turf-clean`
   - Accept --client-slug argument (required)
   - Also accept --dry-run flag to show what would be searched without making API calls
   - Also accept --skip-checked-days N flag (default 30) to skip directories checked within N days

2. Main function `discover_existing_listings(client_slug: str, dry_run: bool, skip_days: int)`:
   - Load Supabase client via existing pattern
   - Fetch client from clients table by slug, then fetch client_profile by client_id
   - If no profile found, exit with error "No client profile found. Run seed_client_profiles.py first."
   - Fetch all enabled directories from directories table
   - For each directory with a submission that was already checked within skip_days, skip it (check submissions table for existing row with notes containing "Discovered via Brave" and updated_at within skip_days)
   - Show pre-run summary: "Checking N directories for {business_name} ({city}). Estimated Brave queries: N"

3. Search loop:
   - For each directory, build query: `site:{domain} "{business_name}" "{city}"`
   - Call search_brave(query=query, client_id=client_id, count=3)
   - If result.get("blocked"): print budget exceeded message, stop discovery, report partial results
   - If web_results found: upsert into submissions table with:
     * client_id, directory_id
     * status: 'existing_needs_review'
     * live_url: first result URL
     * notes: f"Discovered via Brave Search: {result_title}"
     * on_conflict='client_id,directory_id' (don't overwrite if row exists with different status)
   - If no results: log "Not found on {directory_name}" (no submission row created)
   - Rate limit: time.sleep(1.1) between queries (Brave API 1 req/sec limit)

4. Post-run summary:
   - Print: "Discovery complete for {business_name}"
   - Print: "Found on: {N} directories" with list
   - Print: "Not found on: {N} directories"
   - Print: "Skipped (already checked): {N}"
   - Print: "Brave queries used: {N}"

IMPORTANT: Only upsert if no existing submission row exists OR existing row is still 'pending'. If a row exists with status other than 'pending' (e.g., 'submitted', 'verified', 'skipped'), do NOT overwrite it. Use the Supabase upsert with ignoreDuplicates or check before insert.

Use dotenv, existing _get_supabase() pattern, and argparse for CLI.

<task type="auto">
  <name>Task 2: Build CAPTCHA audit script with automated detection</name>
  <files>scripts/seo_engine/captcha_audit.py</files>
  <action>
Create `scripts/seo_engine/captcha_audit.py` -- a lightweight script that fetches each directory's submission URL and checks for known CAPTCHA markers.

Script structure:
1. CLI interface: `python3 -m scripts.seo_engine.captcha_audit`
   - Accept --update flag to write results to Supabase (default: dry-run report only)
   - Accept --only-unknown flag to only check directories with captcha_status='unknown'

2. Main function `audit_captchas(update: bool, only_unknown: bool)`:
   - Fetch all enabled directories (or only unknown ones if flag set) that have a submission_url
   - For each directory, fetch the submission URL with requests.get() with a 10-second timeout and a real User-Agent header
   - Check the HTML response for CAPTCHA markers:
     * reCAPTCHA v2: look for "google.com/recaptcha", "g-recaptcha", "recaptcha" in page source
     * reCAPTCHA v3: look for "recaptcha/api.js?render=" (invisible)
     * hCaptcha: look for "hcaptcha.com", "h-captcha"
     * Cloudflare Turnstile: look for "challenges.cloudflare.com/turnstile"
     * No CAPTCHA: none of the above found
   - Classification logic:
     * No markers found -> 'no_captcha'
     * reCAPTCHA v2 checkbox found -> 'simple_captcha'
     * reCAPTCHA v3 / hCaptcha / Turnstile / image challenge -> 'advanced_captcha'
     * Request failed (timeout, 403, etc.) -> keep as 'unknown', note the error
   - Rate limit: time.sleep(2) between requests (be polite to directory servers)

3. Output:
   - Print a table: Directory | URL | Status | Detection Method
   - Color-code output if terminal supports it (green/yellow/red)
   - Summary: "No CAPTCHA: N | Simple: N | Advanced: N | Unknown: N | Failed: N"

4. If --update flag:
   - For each successfully classified directory, update: supabase.from('directories').update({ 'captcha_status': status, 'captcha_checked_at': datetime.utcnow().isoformat() }).eq('id', directory_id)
   - Print "Updated N directories in Supabase"

5. Known limitation documented in script: This only detects CAPTCHA markers in the initial HTML. Some directories load CAPTCHAs via JavaScript after page load (invisible reCAPTCHA v3, dynamically injected challenges). Those will be classified as 'no_captcha' by this script but may actually have CAPTCHAs. Brian's manual review of the ~15-20 directories classified as 'no_captcha' is the final check before Phase 9 automation. Flag this clearly in output.

IMPORTANT: Use requests library (already in project). Do NOT use Playwright or Selenium for this script -- it's a quick HTTP check, not a full browser render. Directories that require JS rendering for CAPTCHA detection should remain 'unknown' for manual review.
