Create three components following the ClientManager.tsx pattern (inline styles, CSS variables, 'use client', useState, supabase browser client).

1. `src/components/directories/ClientProfileForm.tsx`:
   - Props: `{ clientId: string }` -- loads profile for this client from client_profiles table
   - On mount, fetch the client_profile row for clientId. If none exists, show "No profile found -- run seed script first."
   - Editable form fields grouped in sections:
     * NAP: business_name (required), phone, email, website
     * Address: street, city, state, zip
     * Descriptions: description (textarea), short_description (textarea)
     * Services: services (tag input -- type and press Enter to add, click X to remove)
     * Certifications: certifications (same tag input pattern)
     * Payment methods: payment_methods (same tag input pattern)
     * Hours: hours (simple key-value pairs for each day, e.g., "Monday": "8:00 AM - 5:00 PM")
     * Social: social_links (key-value pairs, e.g., "facebook": "https://...")
     * Other: year_established, logo_url
   - Save button calls supabase.from('client_profiles').update({...}).eq('client_id', clientId)
   - Show "Saved!" confirmation for 2 seconds after successful save
   - Use inline styles with CSS variables: var(--bg-surface) for card background, var(--border) for inputs, var(--accent) for save button, var(--text-primary) and var(--text-secondary) for text

2. `src/components/directories/DirectoryManager.tsx`:
   - No props needed -- loads all directories from Supabase
   - On mount, fetch all directories ordered by tier ASC, name ASC
   - Display as a table/list with columns: Name, Domain, Tier, Trades, DA, CAPTCHA, Enabled, Actions
   - "Add Directory" button at top toggles an inline form (same pattern as ClientManager.tsx add form)
   - Add form fields: name, domain, submission_url, tier (select: 1/2/3), trades (tag input), da_score, submission_method (select), notes
   - Each row renders a DirectoryRow component
   - Filter controls: filter by tier (All / Tier 1 / Tier 2 / Tier 3), filter by CAPTCHA status

3. `src/components/directories/DirectoryRow.tsx`:
   - Props: `{ directory: Directory, onUpdate: () => void }`
   - Display mode: name, domain (as link), tier badge, trades, DA score, CAPTCHA badge, enabled toggle
   - CAPTCHA badge colors per locked decision:
     * no_captcha: green background (var(--success) or #10B981)
     * simple_captcha: yellow/amber (#F59E0B)
     * advanced_captcha: red (var(--danger) or #FF3D57)
     * unknown: grey (var(--text-secondary))
   - Edit mode (toggle): inline edit for tier, submission_url, captcha_status, da_score, trades, notes
   - Enable/disable toggle: updates enabled field directly
   - Save calls supabase.from('directories').update({...}).eq('id', directory.id)

CRITICAL STYLE RULES:
- Inline styles ONLY. No Tailwind classes.
- Use CSS variables from globals.css: var(--bg-surface), var(--bg-depth), var(--border), var(--accent), var(--text-primary), var(--text-secondary), var(--font-mono), var(--font-sans)
- Follow hover pattern from existing components: onMouseEnter/onMouseLeave manipulating e.currentTarget.style
- Default export for each component
