```python
def execute_geo_upgrade(website_path, filename, upgrades, dry_run=True):
```

**Protected pages check:** Block `index.html` -- return `{"status": "blocked", "reason": "protected page"}`.

**File existence check:** Return error if file not found.

**Upgrade execution:** Read the raw HTML as a string. For each upgrade in the `upgrades` array, apply using STRING-LEVEL insertion (not BeautifulSoup serialization -- per research Open Question 2, str(soup) alters whitespace/attributes). Use BeautifulSoup ONLY for reading/detection, then do string manipulation for insertion:

1. **answer_block**: Find the H2 tag containing `after_heading` text (case-insensitive search). Locate the closing `</h2>` tag after that match. Insert the `content` string (should be an HTML paragraph with class="answer-capsule") immediately after `</h2>` with a newline. If heading not found, log warning and skip this upgrade.

2. **stats_injection**: Find the H2/H3 tag containing `target_section` text. Find the next `</p>` tag after that heading (end of first paragraph in that section). Insert the `content` wrapped in `<p>` tags after that `</p>`. If section not found, skip.

3. **freshness_update**: Find the first `<main` or `<article` or `<div class="content` tag. Insert `<p class="freshness-date">{content}</p>` after the opening tag. If none found, insert after the first `<body` tag's `>`. If a `.freshness-date` element already exists, replace its content instead of adding a new one.

**Validation:** Track which upgrades were applied. If none applied, return `{"status": "no_changes", "upgrades_attempted": len(upgrades)}`.

**Write + Git:** If not dry_run and changes were made, write the modified HTML back to the file. Git add, commit with message `[SEO-AUTO] GEO content upgrade: {filename}`, and push. Use the same subprocess git pattern as `_git_commit_push_schema` in seo_loop.py.

Return dict: `{"status": "published"|"generated"|"no_changes"|"blocked"|"error", "upgrades_applied": N, "upgrades_skipped": M, "filename": filename}`.

Include a helper `_find_heading_position(html, heading_text)` that does case-insensitive search for an H2/H3 containing the text and returns the position of the closing tag. Use regex: `re.search(r'<h[23][^>]*>.*?' + re.escape(heading_text) + r'.*?</h[23]>', html, re.IGNORECASE | re.DOTALL)`.

Logic (per research Pattern 5):
- Parse HTML with BeautifulSoup
- Find all H2 tags
- For each H2, check if text ends with "?" OR starts with a question word (how, what, why, when, where, is, can, do, does, should, will, are) using `re.match(r'^(how|what|why|when|where|is|can|do|does|should|will|are)\b', text, re.I)`
- For matching H2s, collect the next 1-2 sibling `<p>` tags as the answer (stop at next H2/H3)
- Return list of `{"question": text.rstrip("?") + "?", "answer": " ".join(answer_texts)}`
- Skip if answer is empty

```python
# Auto FAQ schema detection + injection
if action_type in ("blog_post", "newsjack_post", "location_page", "geo_content_upgrade") and website_path:
    try:
        from .schema_injector import detect_faq_candidates, inject_faq_schema
        # Determine which file was created/modified
        if action_type in ("blog_post", "newsjack_post"):
            target_file = Path(website_path) / "blog" / f"{action.get('slug', '')}.html"
        elif action_type == "location_page":
            target_file = Path(website_path) / "areas" / f"{action.get('slug', '')}.html"
        elif action_type == "geo_content_upgrade":
            target_file = Path(website_path) / action.get("filename", "")

        if target_file and target_file.exists():
            html = target_file.read_text()
            qa_pairs = detect_faq_candidates(html)
            if qa_pairs:
                updated_html = inject_faq_schema(html, qa_pairs)
                if updated_html != html and not dry_run:
                    target_file.write_text(updated_html)
                    print(f"  [faq-auto] Injected FAQ schema ({len(qa_pairs)} Q&As) into {target_file.name}")
                elif updated_html != html:
                    print(f"  [faq-auto] Would inject FAQ schema ({len(qa_pairs)} Q&As) into {target_file.name}")
    except Exception as e:
        print(f"  [faq-auto] FAQ auto-detect failed (non-fatal): {e}")
```
