  <behavior>
    - Given a submission with status='submitted' and submitted_at 8 days ago, verify_submissions queries Brave Search with site:{domain} "{business_name}" "{city}"
    - Given Brave returns web results, the submission updates to status='verified' with live_url set to the first result URL and verified_at set
    - Given Brave returns no results, the submission stays as status='submitted' with metadata.last_verification_check updated
    - Given a submission submitted 15 days ago still unverified, an alert line prints with directory name and client name
    - Given a submission submitted 22 days ago still unverified, status updates to 'needs_review' with explanatory notes
    - Given search_brave returns {"blocked": True}, the script stops checking remaining submissions gracefully
    - Given metadata.last_verification_check was set 3 days ago, the submission is skipped (not re-checked within 7 days)
    - Given --dry-run flag, no Brave API calls are made and no Supabase updates occur
    - Given --client-slug, only that client's submissions are checked; omitting checks all clients with submitted directories
  </behavior>
  <action>
    Create `scripts/seo_engine/verify_submissions.py` following the exact structure of discovery.py:

    1. **Imports and setup**: Same as discovery.py -- argparse, dotenv, supabase create_client, search_brave import. Add datetime/timedelta.

    2. **_get_supabase()**: Identical helper to discovery.py.

    3. **get_clients_with_submissions(sb)**: Query submissions table for distinct client_ids where status='submitted'. If --client-slug provided, resolve to single client_id via clients table.

    4. **get_submissions_to_verify(sb, client_id)**: Query submissions where status='submitted' AND submitted_at <= 7 days ago. Filter out rows where metadata.last_verification_check is within 7 days (do this in Python after fetch since JSONB filtering is awkward in supabase-py).

    5. **verify_single_submission(sb, submission, directory, profile, dry_run)**:
       - Build query: `site:{directory.domain} "{profile.business_name}" "{profile.address_city}"`
       - If dry_run: print query and return "skipped"
       - Call search_brave(query=query, client_id=str(submission.client_id), count=3)
       - If result.get("blocked"): return "budget_exceeded"
       - If web_results found: update submission to verified with live_url and verified_at, return "verified"
       - If no results: update metadata.last_verification_check timestamp (read-then-merge pattern from submission_engine.py), return "not_found"

    6. **escalate_stale_submissions(sb, client_id, dry_run)**:
       - Query submissions where status='submitted' AND submitted_at <= 21 days ago. Update to needs_review with notes. (VER-04)
       - Query submissions where status='submitted' AND submitted_at between 14 and 21 days ago. Print alert with directory name lookup. (VER-03)
       - Process 21-day BEFORE 14-day to avoid alerting on rows already marked needs_review.
       - If dry_run: print what would happen but do not update.

    7. **verify_submissions(client_slug, dry_run)**: Main orchestrator.
       - Get client list (single or all)
       - For each client: load profile, load directories (as lookup dict by id), get submissions to verify
       - Run verify_single_submission for each eligible submission. Stop on budget_exceeded.
       - Run escalate_stale_submissions for each client.
       - Print summary: N checked, N verified, N not_found, N escalated, N needs_review.

    8. **main() + CLI**: argparse with --client-slug and --dry-run flags. Match discovery.py pattern exactly.

    Key constraints:
    - Use search_brave() from brave_client.py for ALL Brave queries (budget gating)
    - Use read-then-merge for metadata JSONB updates (pattern from submission_engine.py)
    - Strip timezone from Supabase timestamps before comparison (replace(tzinfo=None))
    - Add 1-second delay between Brave API calls (time.sleep(1)) to avoid rate limiting
    - Handle empty profile gracefully (skip client with warning, do not crash)
  </action>
