# Plan 13-02: Dashboard Full Keyword History

## Objective

Update the SEO tab to show complete keyword position history from gsc_queries, so sparklines and expanded charts display data from the earliest available date (not just the last few days when a keyword happened to appear in the top 25).

## Tasks

<task id="1" title="Remove 90-day limit on gsc_queries fetch">
In `src/app/page.tsx`, the GSC queries fetch currently loads based on the latest report. Update to fetch ALL gsc_queries for the client with no date limit:

```typescript
const { data: allGscQueries } = await supabase
  .from('gsc_queries')
  .select('query, run_date, position, impressions, clicks')
  .eq('client_id', activeClient.id)
  .order('run_date', { ascending: true });
```

Pass this full dataset to SeoTab as a new prop (e.g., `gscHistory`).
</task>

<task id="2" title="Update SeoTab to use full history for sparklines" depends_on="1">
In `src/components/tabs/SeoTab.tsx`, the keyword table already aggregates gsc_queries by query and builds position history arrays. With the new full dataset:

1. The aggregation logic (lines 86-134) should use the full `gscHistory` prop
2. Each keyword's position array will now contain ALL daily data points from the earliest date
3. The sparkline component already handles variable-length data arrays -- no SVG changes needed
4. The "movement" delta calculation should still compare latest vs previous day

The key change: instead of only having data from days when the keyword was in the top 25, we now have data from EVERY day it appeared in GSC results.
</task>

<task id="3" title="Update expanded chart to show full date range" depends_on="2">
When a keyword row is expanded to show the full LineChart:

1. The X-axis should span from the keyword's first appearance to today
2. Add date labels at reasonable intervals (weekly ticks for >30 days of data)
3. Maintain the inverted Y-axis (lower position = higher on chart)
4. If a keyword has gaps (days it wasn't in GSC), connect the line across gaps rather than showing breaks

The expanded chart should clearly show the trajectory -- e.g., "pressure washing san diego" going from position 20 in late February to position 12 now.
</task>

<task id="4" title="Add date range indicator to keyword table header">
Add a subtle date range indicator showing how far back the data goes:

```
Top Keywords by Position (since Feb 23)
```

This sets expectations -- the user knows the sparklines represent 3 weeks of data, not 3 days.
</task>

## Verification

<must_haves>
- Keyword sparklines show data from the earliest available date in gsc_queries
- Expanded keyword charts show the full position trajectory over all available dates
- "pressure washing san diego" for Integrity Pro shows data going back to Feb 23 (not just Mar 6)
- No performance degradation from loading larger dataset (pagination if needed for clients with 500+ unique queries)
- Date range indicator visible in table header
</must_haves>
