# Plan 13-01: Full GSC Query Storage + Backfill

## Objective

Modify the GSC data pipeline to store ALL queries returned by GSC daily (not just top 25), using upsert logic instead of delete+insert so historical data is never lost.

## Tasks

<task id="1" title="Add unique constraint to gsc_queries">
Add a UNIQUE constraint on (client_id, run_date, query) to gsc_queries table via Supabase migration. This enables upsert logic and prevents duplicates.

```sql
ALTER TABLE gsc_queries ADD CONSTRAINT gsc_queries_client_date_query_unique
  UNIQUE (client_id, run_date, query);
```

Drop the report_id foreign key dependency -- queries should be tied to client+date, not a specific report run.
</task>

<task id="2" title="Modify pull_gsc to fetch all queries" depends_on="1">
In `scripts/run_reports.py`, modify the `pull_gsc()` function:

1. Change `rowLimit` from 25 to 5000 (GSC API max is 25000, 5000 covers any realistic daily query set)
2. Keep the same return structure but with all queries instead of top 25

Current (line 172):
```python
"rowLimit": 25,
```

Change to:
```python
"rowLimit": 5000,
```
</task>

<task id="3" title="Change push_to_supabase to upsert GSC queries" depends_on="2">
In `push_to_supabase()`, replace the DELETE+INSERT pattern for gsc_queries with an upsert:

Current pattern (lines 761-776):
```python
sb.table("gsc_queries").delete().eq("report_id", report_id).execute()
# ... build rows ...
sb.table("gsc_queries").insert(query_rows).execute()
```

New pattern:
```python
query_rows = []
for q in gsc_data.get("top_queries", []):
    query_rows.append({
        "client_id": client_id,
        "report_id": report_id,
        "run_date": run_date,
        "query": q["query"],
        "impressions": q["impressions"],
        "clicks": q["clicks"],
        "position": round(q["position"], 1),
    })

if query_rows:
    sb.table("gsc_queries").upsert(
        query_rows,
        on_conflict="client_id,run_date,query"
    ).execute()
```

This preserves all historical data and updates existing rows on re-run.
</task>

<task id="4" title="Backfill historical GSC data" depends_on="3">
Write a one-time backfill script `scripts/backfill_gsc_history.py` that:

1. For each active client, pull GSC data for every day from their earliest gsc_queries entry to today
2. Use pull_gsc with the new 5000 row limit
3. Upsert all results to gsc_queries

Use 1-day date ranges per API call to get per-day granularity. Add a small delay between calls to respect rate limits.

GSC API retains ~16 months of data, so we can backfill significantly further than what we have.
</task>
