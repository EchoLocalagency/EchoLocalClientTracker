**1. outcome_logger.py -- Add get_directory_summary() function:**

Add a new function `get_directory_summary(client_id, client_trades=None)` after the existing `get_action_history()` function. It must:
- Use `_get_sb()` to get the Supabase client (same as other functions in this file)
- Query `submissions` table with `count="exact"` and `.eq("client_id", client_id).in_("status", ["submitted", "approved", "verified"])` for submitted count. CRITICAL: Use `.in_()` not `.eq("status", "submitted")` because verified/approved submissions should still count as submitted (Pitfall 3 from research).
- Query `submissions` table with `count="exact"` and `.eq("client_id", client_id).eq("status", "verified")` for verified count.
- For total_eligible denominator: Query `directories` table with `.eq("enabled", True)` and `count="exact"`. If `client_trades` is provided and non-empty, filter to directories whose trades overlap with client_trades OR have empty trades (universal directories). If no client_trades, just count all enabled directories.
- Return dict: `{"submitted": int, "verified": int, "total_eligible": int}`
- Guard all queries with try/except returning `{"submitted": 0, "verified": 0, "total_eligible": 0}` on failure (non-fatal)

**2. brain.py -- Add directory_summary section to _build_prompt() and call_brain():**

- Add `directory_summary=None` kwarg to `_build_prompt()` signature (after `paa_gaps=None`)
- Add section in function body (after the existing geo/paa sections, before the final return). Use the if-guard pattern:
  ```
  if directory_summary:
      submitted = directory_summary.get("submitted", 0)
      verified = directory_summary.get("verified", 0)
      total = directory_summary.get("total_eligible", 0)
      prompt += "DIRECTORY COVERAGE:\n"
      prompt += f"  {submitted}/{total} directories submitted | {verified}/{submitted if submitted else total} verified live\n"
      prompt += "  Do NOT propose directory_submission actions -- that is handled by submission_engine.py separately.\n"
      prompt += "\n"
  ```
- Add `directory_summary=None` kwarg to `call_brain()` signature (after `paa_gaps=None`, before `dry_run=True`)
- CRITICAL: Add `directory_summary=directory_summary` to the `_build_prompt()` call inside `call_brain()`. The current call at line 543-550 passes all kwargs positionally. Add `directory_summary` after `paa_gaps` in that positional call. Alternatively, convert the _build_prompt() call to use keyword arguments for the new parameter to avoid positional errors.
