  <action>
Create `src/app/pipeline/page.tsx` as a 'use client' single-file page component. This is the complete pipeline page. Follow project conventions: ALL inline styles using CSS custom properties (var(--bg-surface), var(--border), var(--accent), var(--text-primary), var(--text-secondary), var(--bg-depth), var(--radius-card), var(--font-mono), var(--font-sans)). NO Tailwind classes. NO separate component files.

**Auth guard (top of component):**
- Import and call `useAuth()` to get `{ profile, loading: authLoading, isAdmin }`
- If `authLoading`, render centered "Loading..." with `color: var(--text-secondary)`
- If `!isAdmin`, render centered "Access denied" with `color: var(--text-secondary)`

**State:**
- `leads: PipelineLead[]` -- all pipeline leads
- `checklistProgress: Record<string, { done: number; total: number }>` -- per-lead checklist counts
- `lastContact: Record<string, string>` -- per-lead latest comm occurred_at
- `loading: boolean` -- data loading state
- `stageFilter: PipelineStage | 'all'` -- default 'all'
- `sortField: SortField` -- default 'created_at' (type: 'contact_name' | 'stage' | 'trade' | 'source' | 'days_in_stage' | 'checklist' | 'last_contact' | 'created_at')
- `sortDir: 'asc' | 'desc'` -- default 'desc'

**Data fetching (useEffect on isAdmin):**
Three parallel queries on mount (use Promise.all):
1. `supabase.from('pipeline_leads').select('*').order('created_at', { ascending: false })` -- all leads
2. `supabase.from('pipeline_checklist_items').select('lead_id, completed')` -- for checklist progress
3. `supabase.from('pipeline_comms').select('lead_id, occurred_at').order('occurred_at', { ascending: false })` -- for last contact

Process checklist data into `Record<string, { done, total }>` by iterating and counting per lead_id.
Process comms data into `Record<string, string>` by taking first (latest) occurred_at per lead_id.

For leads with no checklist items in DB, use `STAGE_CHECKLIST_DEFAULTS[lead.stage].length` as the denominator so the column shows "0/N" (not "0/0").

**Stage summary cards (UI-05):**
- Row of 6 cards using flexbox (gap: 12px, flexWrap: wrap)
- Each card: background var(--bg-surface), border 1px solid var(--border), borderRadius var(--radius-card), padding 16px 20px, flex 1, minWidth 120
- Stage label: fontSize 10, fontFamily var(--font-mono), color var(--text-secondary), textTransform uppercase, letterSpacing 0.05em
- Count: fontSize 28, fontWeight 700, marginTop 4
- Use `PIPELINE_STAGES.map()` for consistent ordering
- Derive counts from `leads` array using useMemo

**Stage filter dropdown (UI-03):**
- A select element above the table: options are "All Stages" + each stage from PIPELINE_STAGES
- Style: background var(--bg-depth), color var(--text-primary), border 1px solid var(--border), borderRadius 6, padding 6px 12px, fontSize 12, fontFamily var(--font-mono)

**Pipeline table (UI-02):**
- Full-width table with columns: Name, Stage, Trade, Source, Days, Checklist, Last Contact
- Table style: width 100%, borderCollapse collapse, background var(--bg-surface), borderRadius var(--radius-card), overflow hidden
- Header cells: clickable for sorting (SortHeader pattern from research). Active sort column highlighted with var(--accent). Show arrow indicator (up/down unicode).
- Header style: padding 10px 12px, textAlign left, fontSize 11, fontWeight 600, fontFamily var(--font-mono), textTransform uppercase, letterSpacing 0.05em, cursor pointer, userSelect none, borderBottom 1px solid var(--border)
- Body rows: padding 10px 12px, borderBottom 1px solid var(--border), fontSize 13
- Row hover: background rgba(255,255,255,0.02)

**Column rendering:**
- Name: `lead.contact_name` (with `lead.company_name` in small text below if present)
- Stage: `<select>` dropdown with all PIPELINE_STAGES as options. Styled with background var(--bg-depth), color var(--text-primary), border 1px solid var(--border), borderRadius 6, padding 4px 8px, fontSize 12, fontFamily var(--font-mono). onChange triggers `changeStage()` (UI-04).
- Trade: `lead.trade || '--'`
- Source: `lead.source || '--'`
- Days: computed `daysInStage(lead)` -- `Math.floor((Date.now() - new Date(lead.stage_entered_at).getTime()) / 86400000)`
- Checklist: `checklistProgress[lead.id] ? checklistProgress[lead.id].done + '/' + checklistProgress[lead.id].total : '0/' + STAGE_CHECKLIST_DEFAULTS[lead.stage].length`
- Last Contact: `lastContact[lead.id] ? new Date(lastContact[lead.id]).toLocaleDateString() : '--'`

**Sorting logic (UI-03) -- useMemo:**
- Filter by stageFilter first (skip if 'all')
- Sort by sortField with sortDir. Null-safe: use `(a ?? '') < (b ?? '')` pattern. For numeric fields (days_in_stage, checklist), compare as numbers.
- For 'days_in_stage' sort, compare stage_entered_at timestamps
- For 'checklist' sort, compare done/total ratio
- For 'last_contact' sort, compare occurred_at timestamps (nulls to end)

**Stage change function (UI-04):**
`async function changeStage(leadId: string, currentStage: PipelineStage, newStage: PipelineStage)`:
1. Optimistically update local leads state (update stage AND stage_entered_at to now)
2. `await supabase.from('pipeline_leads').update({ stage: newStage, stage_entered_at: new Date().toISOString(), updated_at: new Date().toISOString() }).eq('id', leadId)` -- if error, log and revert
3. `await supabase.from('pipeline_stage_history').insert({ lead_id: leadId, previous_stage: currentStage, new_stage: newStage })` -- if error, log (don't revert lead update)
4. Update checklistProgress denominator for this lead if stage changed (use STAGE_CHECKLIST_DEFAULTS[newStage].length)

**Page layout:**
- Outer div: padding 32px 40px, maxWidth 1200, margin 0 auto
- Page title: "Pipeline" -- fontSize 24, fontWeight 700, marginBottom 24
- Stage cards row, then filter dropdown (marginTop 20, marginBottom 12), then table
- Empty state when no leads: centered text "No pipeline leads yet" in var(--text-secondary)
- Loading state: centered "Loading pipeline..." in var(--text-secondary)
