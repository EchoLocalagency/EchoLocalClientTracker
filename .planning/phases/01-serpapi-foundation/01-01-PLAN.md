1. Create `supabase_migration_serpapi_usage.sql` in the project root (matches existing migration file pattern like `supabase_migration_v2.sql`):
   - Table `serpapi_usage` with columns: id (UUID, gen_random_uuid), client_id (UUID, references clients(id) ON DELETE CASCADE), query (TEXT NOT NULL), search_type (TEXT NOT NULL DEFAULT 'organic'), location (TEXT), searched_at (TIMESTAMPTZ DEFAULT now()), created_at (TIMESTAMPTZ DEFAULT now())
   - Index `idx_serpapi_usage_client_month` on (client_id, searched_at)
   - Index `idx_serpapi_usage_month` on (searched_at)

2. Run the migration against Supabase using the Supabase CLI or the REST API. If CLI is not available, use the Supabase SQL Editor via the management API or instruct to run manually.

3. Create `scripts/seo_engine/serpapi_client.py` with these functions:

   **Constants:**
   - CLIENT_MONTHLY_LIMIT = 200
   - GLOBAL_MONTHLY_LIMIT = 950

   **`_get_supabase()`** -- returns Supabase client (same pattern as seo_loop.py: `create_client(os.getenv("SUPABASE_URL"), os.getenv("SUPABASE_KEY"))`)

   **`check_budget(client_id: str) -> dict`**:
   - Query serpapi_usage table: COUNT rows where client_id matches AND searched_at >= first day of current month (use `date.today().replace(day=1).isoformat()`)
   - Query serpapi_usage table: COUNT all rows where searched_at >= first day of current month (global)
   - Return dict: `{"allowed": bool, "client_used": int, "client_limit": 200, "global_used": int, "global_limit": 950, "reason": str}`
   - If client_used >= 200, return allowed=False with reason "Client cap reached (X/200)"
   - If global_used >= 950, return allowed=False with reason "Global cap reached (X/950)"
   - Otherwise return allowed=True
   - IMPORTANT: Always query Supabase fresh (no caching). The overhead is negligible for 8 keywords per run.

   **`search_google(query: str, client_id: str, location: str, search_type: str = "organic") -> dict`**:
   - Call check_budget(client_id) first. If not allowed, print `[serpapi] BLOCKED: {reason}` and return `{"blocked": True, "reason": reason}`
   - Build params: q=query, location=location, gl="us", hl="en", api_key=os.getenv("SERPAPI_KEY"), engine="google"
   - Execute via `from serpapi import GoogleSearch; search = GoogleSearch(params); results = search.get_dict()`
   - On success, log to serpapi_usage: insert row with client_id, query, search_type, location
   - IMPORTANT: Only log AFTER successful API call. If the call throws, do NOT insert a usage row.
   - Return the full results dict (includes organic_results, related_questions, answer_box, ai_overview, search_metadata)

   **`format_organic_results(serpapi_results: dict) -> list`**:
   - Convert SerpAPI response to legacy format: list of dicts with keys position, title, url (from "link"), description (from "snippet")
   - This enables backward compatibility with research_runner.py's cache format

   **`check_account_balance() -> dict`**:
   - GET `https://serpapi.com/account.json` with params `api_key=os.getenv("SERPAPI_KEY")`, timeout=10
   - Return dict: `{"searches_used": this_month_usage, "searches_limit": plan_searches_left + this_month_usage, "searches_remaining": plan_searches_left, "plan": plan_name}`
   - Handle request errors gracefully (try/except, print error, return empty dict with error key)

   **Install dependency:** Run `pip install google-search-results` (the legacy SerpAPI package, NOT `serpapi`). Import as `from serpapi import GoogleSearch`.
