<objective>
Build a local HTML analyzer that scores client pages for citation-readiness on a 0-5 binary checklist, store scores daily in Supabase, and hook into seo_loop data collection for baseline accumulation.

Purpose: Establishes the "scoring" half of Phase 2. The brain needs to know which pages are citation-ready and which are not. This module reads local HTML files (zero API cost) and scores them on 5 factors: answer blocks, stats density, schema presence, heading structure, and freshness signals. Daily scoring creates trend data for the brain to use in Phase 3.

Output: geo_scores table in Supabase populated daily with per-page citation-readiness scores.

3. Create `scripts/seo_engine/geo_scorer.py` with:

   a. `score_page(html: str) -> dict` -- Returns {"score": int, "factors": {"answer_block": 0|1, "stats_density": 0|1, "schema_present": 0|1, "heading_structure": 0|1, "freshness_signal": 0|1}}

   b. Five internal checker functions per the research doc:
      - `_check_answer_block(soup)` -- Looks for a concise paragraph (30-80 words) in the first 10 paragraphs
      - `_check_stats_density(soup)` -- Uses STATS_PATTERNS regex list (percentages, years, "over N", "rated N", "N out of N", dollar amounts, square footage). Returns 1 if 2+ patterns match. Explicitly excludes phone number patterns (10-digit) and zip codes (5-digit standalone).
      - `_check_schema(html)` -- Regex for `<script type="application/ld+json">` tag presence
      - `_check_headings(soup)` -- Returns 1 if 3+ H2 tags exist
      - `_check_freshness(soup, html)` -- Checks for 2025 or 2026 in text, or "updated"/"last modified"/"published" patterns

   c. `score_all_pages(client_config: dict) -> list` -- Main entry point:
      - Get website_local_path from client_config
      - Glob for *.html files in that directory (non-recursive -- these are flat static sites)
      - For each HTML file: read it, call score_page(), build result dict
      - Get client_id from client_config["_supabase_id"]
      - Get base website URL from client_config["website"]
      - Upsert each score to Supabase geo_scores table using .upsert() with on_conflict="client_id,page_path,scored_at" (so running twice in a day updates rather than duplicates)
      - Return list of {"page_path": str, "score": int, "factors": dict} for logging
      - Use _get_supabase() pattern (inline helper, same as serpapi_client.py)
      - Wrap Supabase operations in try/except, log errors but don't crash

   d. Install beautifulsoup4: `pip install beautifulsoup4`

   Use `from bs4 import BeautifulSoup` with parser "html.parser" (stdlib, no extra install needed).
