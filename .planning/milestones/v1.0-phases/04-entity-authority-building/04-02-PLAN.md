   ```python
   total = c["supporting_count"] + c["gap_count"]
   c["authority_completeness"] = round(c["supporting_count"] / total, 2) if total > 0 else 0.0
   ```
   This is a derived metric -- no new Supabase table needed.

2. In geo_data.py, add get_all_paa_questions(client_id) function:
   - Query serp_features table: select keyword, paa_questions where client_id matches, order by collected_at desc, limit 100
   - Deduplicate by keyword (keep latest only, using seen_keywords set)
   - Collect all unique questions across keywords into a set
   - Handle paa_questions being either a list or JSON string (parse if string)
   - Return list of unique question strings
   - Follow existing _get_supabase() pattern

3. Create scripts/seo_engine/paa_matcher.py with two functions:

   a. extract_page_headings(website_path):
      - Takes Path to website root
      - Collects HTML files: *.html in root, blog/*.html, areas/*.html (same non-recursive glob as geo_scorer)
      - For each file, parse with BeautifulSoup(html, "html.parser")
      - Extract all H2 headings (get_text(strip=True))
      - Also grab first 2000 chars of text content for deeper matching
      - Return list of {"path": relative_path_str, "headings": [str], "text": str}

   b. match_paa_to_content(paa_questions, pages):
      - Takes list of question strings and list of page dicts from extract_page_headings
      - For each question, find best matching H2 heading across all pages using difflib.SequenceMatcher
      - Normalize: lowercase, strip trailing "?"
      - Match threshold: ratio >= 0.6
      - Also check if question text appears as substring in page text (case-insensitive) as a secondary match signal
      - Return {"matched": [{"question": str, "page": str, "heading": str, "score": float}], "gaps": [str]}
      - gaps = questions with no match above threshold

Use only stdlib (difflib, pathlib) + BeautifulSoup4 (already a dependency). No new packages.
